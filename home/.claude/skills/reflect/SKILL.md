description: Learn durable, cross-project preferences from this session. Use at session end, or when the user corrects your behavior in a way that should apply everywhere.
allowed-tools: Read, Edit, Write
---
Review the session for DURABLE, CROSS-PROJECT preferences the user expressed or implied through corrections - things that should shape how you work in *every* repo, not facts about this one project.

Good captures: tooling preferences ("use uv not pip"), style/workflow rules ("always run tests before saying done"), communication preferences, recurring corrections.
Do NOT capture: project-specific facts (those belong in that project's memory), one-off instructions, anything secret, or anything already present.

Append new items as terse bullets to `~/.claude/memory/preferences.md` (create it if missing). Before adding, read the file and skip duplicates or near-duplicates. Keep each bullet a single specific, verifiable line. If nothing durable came up, do nothing.
